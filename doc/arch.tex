\chapter{GNU arch}

The {\tt itools} source code is managed by {\em tla} (also known as
{\em GNU arch}\footnote{\tt http://www.gnu.org/software/gnu-arch/}),
a distributed control version system.

The purpose of this chapter is to help people:

\begin{itemize}
  \item To keep track of the {\tt itools} development.

  \item To contribute to the development of {\tt itools}.
\end{itemize}


\section{Keeping track of {\tt itools}}

If you just want to look at the sources once in while, you can do it
through the web, see my public {\em tla} archive at:

\begin{code}
    http://in-girum.net/cgi-bin/archzoo.cgi
\end{code}

\subsection{Check out}

But if you want to check out {\tt itools} from the archive to be up-to-date
regularly, you will have to install {\em tla} (not covered here) and to set
up it.

\subsubsection{Set your id}

\begin{code}
    $ tla my-id "J. David Ibáñez <jdavid@itaapy.com>"
\end{code}


\subsubsection{Register the official itools archive}

\begin{code}
    $ tla register-archive jdavid@itaapy.com--public \
          http://in-girum.net/~jdavid/archives/public
\end{code}


\subsubsection{Check out itools}

\begin{code}
    $ tla get jdavid@itaapy.com--public/itools--main--0.4 itools
\end{code}

\subsubsection{Update}

\begin{code}
    $ cd itools
    $ tla update
\end{code}

To be done often!
